⚠️ 風險聲明：本系統所有分析與排名結果均僅供資訊整理與研究參考，不構成任何投資建議。股票投資存在市場風險，請使用者審慎評估自身風險承受能力。
🏦
交易核心
透過 Alpaca Markets API 進行全自動美股下單，策略以 JSON 描述，無需修改 Python 代碼即可新增策略。支援多帳戶管理，每帳戶同時只使用一項策略。
📊
視覺化 Dashboard
GitHub Pages 靜態 HTML 儀表板，顯示現金水位、持倉清單、個股 1日/1週/1月 績效、NAV 與回撤走勢，可與 NASDAQ / S&P 500 對比。
📧
全自動通知
每日上午 06:00 Email 發送日報（前十大績效股、P&L、本益比）。交易發生時即時通知。所有程序在 GitHub Actions 雲端執行，不依賴本地機器。

策略 JSON Schema（參考 trading_strategy_schema_v2_complete）
💡 設計原則：新增投資策略時，只需在 strategies/ 資料夾新增一個 JSON 檔案，無需修改任何 Python 代碼。 
{
  "schema_version": "2.0",
  "strategy_id":     "nasdaq_top10_equal_weight",
  "strategy_name":   "NASDAQ 市值前十等權重策略",
  "description":    "每日選取 NASDAQ 市值前 10 大股票，各佔投資組合 10%",
  "enabled":         true,

  // ── 選股條件 ─────────────────────────────────────────
  "universe": {
    "exchange":    "NASDAQ",
    "filter":     "market_cap_top_n",
    "top_n":      10,
    "min_price":  5.0,
    "exclude_etf": true
  },

  // ── 資金配置 ─────────────────────────────────────────
  "allocation": {
    "method":          "equal_weight",
    "weight_per_stock": 0.10,      // 每檔 10%
    "max_positions":   10,
    "share_rounding":  "floor",    // 只買整數股，無條件捨去
    "cash_buffer_pct": 0.02       // 保留 2% 現金緩衝
  },

  // ── 再平衡規則 ────────────────────────────────────────
  "rebalance": {
    "triggers": [
      "monthly_first_trading_day",   // 每月第一個交易日
      "new_deposit"                  // 新資金進入時
    ],
    "drift_threshold_pct": 0.05,   // 偏離 5% 才觸發（避免過度交易）
    "daily_rebalance":    false    // ★ 不每日再平衡
  },

  // ── 買進規則 ──────────────────────────────────────────
  "entry": {
    "order_type":  "market",
    "time_in_force": "day",
    "execution_time": "market_open"
  },

  // ── 賣出規則 ──────────────────────────────────────────
  "exit": {
    "condition":    "dropped_from_universe",  // 跌出前十即賣出
    "stop_loss_pct": -0.20,               // 個股虧損 20% 停損
    "order_type":  "market"
  },

  // ── 通知設定 ──────────────────────────────────────────
  "notifications": {
    "on_trade":    true,    // 交易即時通知
    "daily_report": true,   // 每日報告
    "report_time": "06:00" // 每天上午 06:00 發送
  }
}

✅ PHASE 3 測試案例（全部在 Paper 環境執行）
ID測試項目測試方式驗收標準
P3-01 首次建倉 - 10 檔等權重 $100,000 現金，執行首次買進 買進 10 檔，每檔約 10%（$10,000），均為整數股
P3-02 整數股計算 股價 $173.5，可用 $10,000 計算 57 股（57×$173.5=$9,889.5），不買 58 股
P3-03 跌出前十自動賣出 模擬某股跌出 Top 10，執行交易 該股賣出，新進股買入，持倉維持 10 檔
P3-04 停損觸發 模擬個股虧損達 -20% 自動執行賣出，不再持有該股
P3-05 月初再平衡 持倉偏離 > 5%，執行再平衡 各持倉恢復至 10%，交易次數最小化
P3-06 新資金再平衡 帳戶新增 $20,000，觸發再平衡 按當前 Top 10 重新配置，僅執行必要交易
P3-07 策略切換安全性 Account A 切換策略，Account B 不受影響 兩帳戶策略獨立，互不干擾
P3-08 多帳戶 Workflow 遍歷 執行 daily_trading.yml，兩個帳戶均執行 兩帳戶均產生交易記錄，執行時間 < 10 分鐘
P3-09 不每日再平衡 非月初、非新資金日執行 Workflow 不產生再平衡交易，僅執行選股更換

📅 daily_trading.yml
# 每個交易日市場開盤前 5 分鐘觸發
on:
  schedule:
    - cron: '25 13 * * 1-5'  # UTC 13:25 = 美東 09:25
  workflow_dispatch:  # 允許手動觸發

jobs:
  trading:
    runs-on: ubuntu-latest
    steps:
      - name: 遍歷所有帳戶執行策略
        run: python src/account_runner.py
        # 1. 讀取 accounts.json 帳戶清單
        # 2. 每個帳戶載入對應策略 JSON
        # 3. 選出今日 NASDAQ Top 10
        # 4. 計算買進/賣出清單
        # 5. 執行下單
        # 6. 發送交易通知 Email
📧 daily_report.yml
# 每天上午 06:00 台灣時間（UTC 22:00 前一天）
# 使用 cron-job.org 外部觸發，確保準時
on:
  workflow_dispatch:  # 由 cron-job.org 呼叫

jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - name: 生成日報 JSON Model
        run: python src/report_generator.py
      - name: 儲存歷史報告
        run: git add reports/ && git commit && git push
      - name: 發送 Email（View 渲染）
        run: python src/report_view.py --send-email
      - name: 更新 Dashboard
        uses: actions/deploy-pages@v4

CLAUDE.md — 專案記憶文件（快速上手） 
# Alpaca 全自動美股交易系統 — 專案記憶

## 核心架構
- 所有交易邏輯由 JSON 驅動，strategy_engine.py 不依賴特定策略
- 新增策略：只需在 strategies/ 新增 JSON 檔案
- 多帳戶：accounts.json 定義帳戶清單，每帳戶一個策略
- 報告 MVC：report_generator.py (Model) + report_view.py (View)
- 全雲端：所有程序在 GitHub Actions 執行，不依賴 local machine

## 關鍵規則（必須遵守）
- 每帳戶同時只能使用一個策略（active_strategy 欄位）
- 只買整數股（floor 向下取整）
- 每檔股票投入比例固定 10%
- 不每日再平衡（只有月初或新資金進入才再平衡）
- API Key 存放在 GitHub Secrets，絕不進入 Git 倉庫

## 每日執行流程
1. daily_trading.yml：市場開盤前選出 Top 10，執行買賣
2. daily_report.yml：生成 JSON 報告 → 儲存歷史 → 發 Email → 更新 Dashboard

## GitHub Secrets 需求
- ACC{N}_API_KEY / ACC{N}_SECRET_KEY（每帳戶一組）
- EMAIL_SENDER / EMAIL_PASSWORD / EMAIL_RECIPIENTS
